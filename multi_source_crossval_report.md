## 四、Phase 4: 集合竞价分析 (核心发现)

### 4.1 AKShare 1m 数据结构

以 000001.SZ 为例:

```
时间                 开盘   收盘    最高    最低    成交量      成交额          均价
2026-03-24 09:30:00  0.0   10.52  10.52  10.52   11,715   12,324,180    10.520   ← 集合竞价结果
2026-03-24 09:31:00  0.0   10.56  10.57  10.53   29,673   31,301,516    10.541   ← 连续交易开始
2026-03-24 09:32:00  0.0   10.57  10.59  10.56   32,324   34,178,263    10.555
...
2026-03-25 14:58:00  0.0   10.94  10.94  10.94      113      123,554    10.871
2026-03-25 14:59:00  0.0   10.94  10.94  10.94        0            0    10.871   ← 静默
2026-03-25 15:00:00  0.0   10.94  10.94  10.94    8,386    9,174,284    10.872   ← 收盘集合竞价
```

### 4.2 关键发现

#### 发现 1: `开盘` 列全部为 0.0 (AKShare 1m 缺陷)

| 周期 | 开盘列 | 状态 |
|------|--------|------|
| **1m** | 全部 0.0 | ❌ 数据缺陷 |
| **5m** | 正常 (如 10.52) | ✅ |
| **1d** | 正常 (如 10.52) | ✅ |

**影响**: 使用 AKShare 1m 数据时**不可依赖 open 字段**。若需 1m OHLC 完整数据，应：
- 使用 5m 数据代替 (open 正常)
- 或用前一 bar 的 close 近似当前 bar 的 open

#### 发现 2: 09:30 首Bar = 集合竞价结果Bar

| 特征 | 值 | 含义 |
|------|-----|------|
| 时间 | 09:30:00 | 交易日第一根Bar |
| close = high = low | 10.52 | 集合竞价成交价 (= 日K开盘价 10.52) |
| volume | 11,715 | 集合竞价撮合量 |

**证据**:
- 09:30 bar 的 close/high/low 三价合一 = 日K开盘价 → 这就是竞价撮合价
- 无 09:15-09:25 独立竞价 bar (0 根)
- 无 09:25-09:30 静默期 bar (0 根)

#### 发现 3: 首Bar量/后续均量比值分析

| 品种 | 首Bar量 | 后续均量 | 比值 | 解读 |
|------|---------|---------|------|------|
| 000001.SZ 平安银行 | 11,715 | ~19,800 | 0.59x | 集合竞价量 < 连续交易均量 |
| 000002.SZ 万科A | 14,948 | ~14,800 | 1.01x | 竞价量 ≈ 连续均量 |
| 000858.SZ 五粮液 | 728 | ~2,350 | 0.31x | 竞价量远低 |
| 000651.SZ 格力电器 | 1,545 | ~5,700 | 0.27x | 竞价量远低 |
| 000333.SZ 美的集团 | 2,821 | ~3,400 | 0.83x | 竞价量接近 |
| 002594.SZ 比亚迪 | 10,505 | ~21,400 | 0.49x | 竞价量较低 |
| 300750.SZ 宁德时代 | 5,133 | ~7,900 | 0.65x | 竞价量较低 |
| 002142.SZ 宁波银行 | 798 | ~2,350 | 0.34x | 竞价量远低 |
| 600519.SH 贵州茅台 | 235 | ~550 | 0.43x | 竞价量远低 |
| 601318.SH 中国平安 | 4,064 | ~12,300 | 0.33x | 竞价量较低 |

**结论**: 首Bar 量/后续均量比值全部 ≤ 1.01x，大部分 < 0.6x。这表明:
- **AKShare 的 09:30 首Bar 仅包含集合竞价撮合量**，不包含 09:31 的连续交易量
- 竞价量通常只有连续交易单分钟均量的 **0.3x ~ 0.6x**
- 不存在"竞价量膨胀首Bar"问题

#### 发现 4: 收盘集合竞价 (14:57-15:00)

每日有 4 根 bar 覆盖收盘竞价区间 (14:57, 14:58, 14:59, 15:00):
- **14:59 bar**: volume = 0 (静默分钟，无成交)
- **15:00 bar**: volume = 8,386 (收盘竞价全部撮合量)
- 格式与开盘竞价一致: 撮合量集中在一根 bar

#### 发现 5: 每日 241 根 Bar 的构成

```
09:30        ×  1 根    集合竞价结果
09:31-11:30  × 120 根   上午连续交易
13:00-14:56  × 117 根   下午连续交易 (含13:00)
14:57-15:00  ×  3 根    收盘竞价区间 (14:57, 14:58含数据; 14:59=0; 15:00=竞价)
合计: 241 根/日
```

## 七、对量化策略的影响 & 建议

### 7.1 数据源选择建议

| 场景 | 推荐数据源 | 原因 |
|------|-----------|------|
| A股分钟回测 | AKShare 5m | open 正常, 收敛性好 |
| A股日线回测 | AKShare 1d 或 DAT 1d | 两者一致 |
| 精确 VWAP 计算 | AKShare 5m (非1m) | 1m open=0 无法用 |
| 集合竞价分析 | Tick (待修复崩溃) | 仅 Tick 有完整竞价逐笔 |
| 期货回测 | QMT DAT (需下载数据) | AKShare 不支持期货分钟 |
| 实盘监控 | xtdata 实时API | 另行评估 |

### 7.2 代码层面注意事项

```python
# 1. AKShare 1m 数据不要使用 open 列
df_1m = load_akshare_data(sym, "1m")
# ❌ df_1m["open"]  → 全是 0.0
# ✅ df_1m["close"], df_1m["high"], df_1m["low"], df_1m["volume"]  → 正常

# 2. 识别集合竞价Bar (AKShare 1m)
# 09:30 bar: close = high = low = 开盘价, volume = 竞价撮合量
# 15:00 bar: volume = 收盘竞价量

# 3. 241 bars/day 而非 240 bars/day
# 如果策略假设每日固定240根1m bar，需调整逻辑

# 4. AKShare 5m 首Bar时间为 09:35 (不是09:30)
# 5m bar采用右端标记: 09:35表示09:30-09:35区间
```

### 7.3 DAT 数据修复建议

当前 `userdata_mini` 目录数据严重不足，需要:
1. 在 QMT 客户端执行"历史数据下载"
2. 或检查是否有其他 `datadir` 目录包含完整数据
3. 期货数据需确认主力合约代码是否正确 (XX01 格式)
