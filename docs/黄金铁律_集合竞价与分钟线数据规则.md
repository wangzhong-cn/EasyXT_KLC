> **期货夜盘（时间 ≥ 18:00）的分钟K线，必须归属到下一个交易日，

### 5.1 掘金量化 (MyQuant)

**文档原文**:
- `context.data` 示例显示首根bar: `bob: 2020-12-21 09:30:00+08:00`, `eob: 2020-12-21 09:31:00+08:00`, `volume: 114365`

**结论**: ✅ 首bar = 09:30，竞价数据被吸收

### 5.2 天勤量化 (TqSdk/Shinnytech)

**文档原文**:
- K线 `datetime` 字段含义: **"K线起点时间(按北京时间)"**
- `trading_time` 字段示例(铜期货):
  ```python
  'day': [['09:00:00', '10:15:00'], ['10:30:00', '11:30:00'], ['13:30:00', '15:00:00']]
  'night': [['21:00:00', '25:00:00']]  # 25:00 = 次日01:00
  ```
- 连续竞价从 09:00 / 21:00 开始，竞价时段(08:55-09:00 / 20:55-21:00)不在`trading_time`范围内

**结论**: ✅ 连续竞价起点在竞价结束后，竞价成交被吸收进首bar

### 5.3 聚宽 (JoinQuant)

- 提供独立的 `get_call_auction()` API 获取"集合竞价时的tick数据"
- 常规 `get_price()` 返回的分钟bar不包含独立竞价bar

**结论**: ✅ 竞价数据需专门API，常规bar中竞价被吸收

### 5.4 Tushare

- `pro_bar` 接口支持 `freq='1min'`，返回标准OHLCV数据
- 文档中无任何关于独立竞价bar的描述

**结论**: ✅ 标准分钟bar，竞价被吸收

### 6.2 夜盘交易日归属代码

```python
import bisect
from datetime import time as dtime

def assign_trading_day(bar_timestamp, sorted_trading_days, has_night_session):
    """将分钟bar的时间戳分配到正确的交易日"""
    if has_night_session and bar_timestamp.time() >= dtime(18, 0):
        cur = pd.Timestamp(bar_timestamp.date())
        idx = bisect.bisect_right(sorted_trading_days, cur)
        if idx < len(sorted_trading_days):
            return sorted_trading_days[idx]
        return cur  # fallback: 不应发生
    return pd.Timestamp(bar_timestamp.date())
```

### 6.3 数据质量检查清单

- [ ] `sum(1m volume)` == `1D volume`（按交易日正确归属后）
- [ ] 首根bar时间符合预期（A股09:30 / 商品09:01 / 夜盘21:01）
- [ ] 不存在09:25/08:55/20:55时间戳的独立bar
- [ ] 5m聚合 == 1D（0%残差，最可靠验证）
- [ ] 每日bar数量在合理范围内

## 八、附录：常见误区澄清

### Q1: "商品期货是否存在二次集合竞价？"

**不存在。** 夜盘集合竞价(20:55-21:00)和日盘集合竞价(08:55-09:00)是两个独立的开盘竞价，分别对应夜盘开盘和日盘开盘。它们不是"二次竞价"，而是两个独立交易时段各自的开盘竞价。并且在QMT数据中，两个竞价的成交量分别被吸收进各时段的首根常规bar（21:01和09:01）。

### Q2: "为什么商品1m残差不是精确0？"

**v2 API版本中确实观察到+0.14%微偏差**，但v4 DAT直读版已证实这是API层时间截断精度问题，不是底层数据错误。v4直接读取DAT二进制文件，商品期货1m→1d残差也是**精确0.000000%**（12组全部100%精确匹配）。

### Q3: "stock的vol_match%为什么不是100%？"

stock S2(1m→1d)平均99.68%、S3(5m→1d)99.95%的match率，差异来自首尾交易日的不完整数据（验证脚本已排除首末日但仍有微量边界效应）。**volume残差 = 0.000000%** 证明实际数据完全正确。

### Q4: "QMT的1m bar时间戳到底是bar起始还是bar结束？"

- **DAT底层数据**: 统一使用 **bar起始时间**（A股09:30, 商品09:00/21:00）
- **API返回数据**: A股=bar起始时间（09:30）, 商品=bar结束时间（09:01/21:01）
- **差异原因**: API层对商品期货做了+1分钟的时间戳偏移，底层DAT文件无此偏移
- **项目代码处理**: 读API时按API约定处理；读DAT时按DAT约定处理
