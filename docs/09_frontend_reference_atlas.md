## 参考图集提炼出的 6 个稳定结构模式

### 1. 图表居中，交易在侧

参考来源：TradingView / OKX / MT5 风格。

稳定模式：

- 中间是主图
- 右侧是报价/下单/盘口/风险
- 底部是持仓/委托/成交/历史

这应成为 `交易工作台` 的默认布局。

### 2. 市场入口独立成区

参考来源：同花顺 / 东方财富。

稳定模式：

- 左侧是自选 / 搜索 / 标的树 / 板块导航
- 用户先选市场对象，再进入工作区

### 3. 热力图必须独立为路由

参考来源：同花顺云图 / 东方财富行业热区。

稳定模式：

- 热力图是独立分析视图
- 不要与交易主工作台硬塞在同一页

### 4. 观点 / 社区 / 脚本是内容层，不是交易层

参考来源：TradingHero 观点流 / 脚本流。

稳定模式：

- 卡片式信息流
- 点赞 / 评论 / 收藏等轻交互
- 与交易终端分离，避免主工作台过载

### 5. 工作区布局预设是高优先级能力

参考来源：OKX 工作区管理、多布局切换。

稳定模式：

- 标准布局
- 专业布局
- 双图 / 四图
- 自定义布局

### 6. 深色、高密度、可组合面板

参考来源：几乎全部截图。

稳定模式：

- 深色主题
- panel card 化
- splitter / tab / dock 体系化
- 高信息密度但分区清楚

## 路由级蓝图

## Route A：交易工作台（主战场）

### Route A 页面结构

- 顶部：市场导航 / 周期 / 指标 / 工作区切换
- 左侧：自选 / 搜索 / 标的列表
- 中间：K 线图主舞台
- 右侧：交易面板 / 报价 / 风险信息
- 底部：持仓 / 委托 / 成交 / 历史 / 账户

### Route A 参考价值最高的截图

- OKX / TradingView 风格终端
- TradingHero 的图表 + 右侧报价面板
- 东方财富 / 同花顺 的下方持仓与列表区

## Route B：热力图

### Route B 页面结构

- 全屏或近全屏热力图
- 顶部板块/主题切换
- 支持按行业 / 风格 / 指数切换

### Route B 参考价值最高的截图

- 同花顺云图

## Route C：观点 / 社区 / 脚本

### Route C 页面结构

- 卡片流
- 每张卡片包含：图、标题、摘要、互动数
- 支持脚本 / 观点 / 社区三类过滤

### Route C 参考价值最高的截图

- TradingHero 卡片页

## Route D：同步状态 / 系统可观察页

### Route D 页面结构

- SQLite 逻辑序号
- active shard
- DuckDB shadow 版本
- sync_status
- last_good_version
- backup / restore / verify 状态

### Route D 设计目标

- 这页是“新壳不再黑盒”的关键页

## 组件模板清单

### 1. WatchlistPanel

用于：

- 自选列表
- 搜索结果
- 标的切换

必须支持：

- 搜索框
- 价格 / 涨跌展示
- 快速切换标的

### 2. ChartStage

用于：

- K 线主舞台
- 指标/叠加物
- 交易标注与结构标注

必须支持：

- 单图 / 多图布局
- 顶部工具条
- 图表事件桥

### 3. TradePanel

用于：

- 买卖切换
- 数量 / 价格 / 杠杆 / 风险参数
- 快捷下单

### 4. PositionTabs

用于：

- 持仓
- 委托
- 成交
- 历史
- 账户

### 5. HeatmapCanvas

用于：

- 行业 / 板块热区
- 大盘热力图

### 6. IdeaCardGrid

用于：

- 观点
- 社区内容
- 脚本市场

### 7. SystemStatusBoard

用于：

- 同步状态
- backup / restore / verify
- active shard / shadow version

## 禁止回退的旧形态

以下旧模式在新前端中禁止继续扩张：

1. 一个页面承载过多线程行为
2. 图表 / 数据查询 / 后台任务 / 交易状态全混在一个 Qt widget 中
3. GUI 直接拼 SQL 或直接知道过多数据库细节
4. 研究 / 热力图 / 社区内容继续塞进交易主工作台

## 给团队的使用方式

以后评审任何新前端页面时，至少对照这 3 个问题：

1. 这个页面属于哪个路由层级？
2. 它更像交易工作台、热力图、内容页还是系统页？
3. 它是否又把旧 Qt 的耦合模式带进了新壳？

如果答不清，就先回到这份图纸再设计。
