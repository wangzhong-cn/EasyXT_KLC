# ADR 目录说明

> ADR（Architecture Decision Record）用于记录**已经做出的重要架构决策**，而不是记录尚未定案的想法。

- [`ADR-0002-Qt主线退役与迁移预算重分配.md`](ADR-0002-Qt主线退役与迁移预算重分配.md)

## 什么时候应该新增 ADR

当一个决策满足下面任一条件时，建议新增 ADR：

- 会影响跨模块的数据口径或系统边界
- 会影响长期维护成本或回滚策略
- 会成为后续评审、门禁或运维解释的依据
- 需要明确“为什么这样做，而不是那样做”

## 当前约定

ADR 文档应尽量包含以下部分：

1. 背景
2. 决策
3. 非目标
4. 影响
5. 执行与验收
6. 回滚策略
7. 后续动作
