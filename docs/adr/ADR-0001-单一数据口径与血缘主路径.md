# ADR-0001：单一数据口径与血缘主路径

- 状态：Accepted
- 日期：2026-03-10
- 决策人：Data Governance / Quant Platform
- 关联文档：[data_infrastructure_diagnosis_v2.md](file:///d:/EasyXT_KLC/docs/data_infrastructure_diagnosis_v2.md)

## 背景

当前仓库存在两类数据治理表：

- `data_ingestion_status`：新链路，承载血缘字段（`schema_version`、`ingest_run_id`、`raw_hash`、`source_event_time`）。
- `data_governance`：历史对账用途，字段语义与血缘主链路不一致。

若双表长期并行且无主从关系，会导致审计口径分叉，出现“门禁通过但审计解释不一致”的风险。

## 决策

1. `data_ingestion_status` 作为唯一主审计路径。
2. `data_governance` 保留为历史兼容表，不承载主链路治理指标。
3. 所有新写入、回填、周报、门禁解释均以 `data_ingestion_status` 为唯一事实源。
4. 血缘字段最小集合固定为：
   - `schema_version`
   - `ingest_run_id`
   - `raw_hash`
   - `source_event_time`

## 非目标

- 不在本 ADR 中合并/删除 `data_governance` 历史数据。
- 不在本 ADR 中改动历史 GUI 对账逻辑。

## 影响

- 好处：审计口径单一，发布证据链可闭环。
- 代价：需要维护一次迁移与回填脚本执行流程。
- 风险：历史表仍存在，需在文档中持续标注“非主路径”。

## 执行与验收

1. 迁移脚本执行：
   - `python tools/migrate_qmt_db.py --db <DB_PATH> --dry-run`
   - `python tools/migrate_qmt_db.py --db <DB_PATH>`
2. 回填脚本执行：
   - `python tools/lineage_backfill.py --db <DB_PATH> --dry-run`
   - `python tools/lineage_backfill.py --db <DB_PATH>`
3. 门禁校验：
   - `python tools/p0_gate_check.py --strict --json`
   - 验收条件：`strict_gate_pass=true` 且 `schema_version_check=pass`

## 回滚策略

- 如迁移异常，回滚到变更前数据库快照。
- 迁移脚本仅执行幂等 DDL 与字段回填，不删除历史表。

## 后续动作

- 每月治理回顾中固定检查主路径一致性。
- 若未来完全下线历史链路，再起草新的 ADR 说明删除计划。
