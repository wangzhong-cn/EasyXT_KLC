# EasyXT结构量化交易体系底层架构与实操方案
## 一、核心建模与理论体系构建
### （一）理论体系层级架构（人类认知框架下不可撼动）
| 层级 | 核心内容 | 定位 | 不可突破约束 |
|------|----------|------|--------------|
| 0层（科学哲学元规范） | 可证伪性、分层证伪、演绎-证伪、开放体系、主客观隔离 | 认识论地基 | 禁止归纳历史规律预测未来，分层证伪优先级不可颠倒 |
| 1层（数学元公理） | 范畴论（对象与态射、复合、单位态射公理）、代数拓扑、测度论、非遍历动力学 | 数学基础 | 必须保持结构的拓扑不变性，禁止破坏因果律 |
| 2层（物理元公理） | 热力学第二定律、兰道尔原理、开放耗散系统自组织公理、CAS涌现公理 | 物理锚点 | 信息-能量-熵三者严格统一，禁止违背耗散结构演化规律 |
| 3层（核心理论） | 拓扑数据分析（TDA）、突变论、多重分形理论、复杂网络理论 | 逻辑骨架 | 结构定义必须基于拓扑不变量，禁止主观设定结构规则 |
| 4层（数学框架） | 非遍历贝叶斯非参数统计（DPMM、GPR、贝叶斯变点检测）、极值理论（EVT） | 计算引擎 | 参数空间必须动态增长，禁止固定模型结构 |
| 5层（验证体系） | 序贯贝叶斯因子检验、蒙特卡洛拓扑不变性检验 | 自证伪闭环 | 禁止固定样本回测，必须分层校验假设 |
| 6层（算法工具） | 持续同调、时序GNN、回声状态网络、贝叶斯优化 | 工程落地 | 工具必须可解释，禁止黑箱化决策 |
| 7层（约束与开放机制） | 层级隔离、自适应监控、永久开放更新 | 安全护栏 | 禁止上层内容反向修改底层元公理 |

### （二）核心理论核心结论
1. **结构本源定义**：市场价格行为的底层实体是开放耗散复杂自适应系统，其演化路径的拓扑不变量（连通分支、环、空洞）是分析核心，所有可观测维度（价格、时间、成交量）均为结构本源的同胚投影。
2. **核心数学工具适配**：
   - 代数拓扑（持续同调）：定义结构的本质属性，区分有效结构与噪声
   - 突变论：量化结构相变的7种基本类型及边界条件
   - 贝叶斯非参数统计：解决开放体系下的概率推断，支持动态参数空间
   - 多重分形理论：解释结构跨尺度自相似性
3. **体系核心矛盾解决**：
   - 开放与收敛：贝叶斯后验概率分布提供明确指向，新数据持续更新保证开放
   - 有限与无限：有限域观测+拓扑不变量推断，无限逼近公理本质
   - 主观与客观：严格区分结构客观指向与人为预判，所有结论基于数据演绎

## 二、数据处理关键问题（理论落地核心争议）
### （一）除权数据处理方案（硬约束）
#### 1. 核心结论
- 前复权/后复权会破坏价格路径的拓扑不变性与因果律，禁止用于结构分析
- 唯一合法方案：**原始价格 + 局部除权因子映射**（必须遵守的硬约束）

#### 2. 局部除权因子工作原理
| 步骤 | 操作内容 | 核心逻辑 |
|------|----------|----------|
| 1 | 锁定单个结构周期边界 | 仅对当前分析周期生效，不影响全局数据 |
| 2 | 提取周期内除权事件 | 仅使用周期内已发生的除权因子，无未来信息泄露 |
| 3 | 局部线性映射 | 除权前价格 × 映射因子（除权后参考价/除权前收盘价）= 映射后价格 |
| 4 | 保持局部相对涨跌幅不变 | 映射仅改变绝对价格，不改变拓扑结构 |
| 5 | 拼接连续路径 | 除权前后价格无缝衔接，K线连续且真实 |

#### 3. 三种除权处理方案对比
| 方案 | 路径因果性 | K线连续性 | 拓扑不变性 | 可复现性 | 适用场景 |
|------|------------|------------|------------|----------|----------|
| 局部除权因子映射 | 完全符合（无未来信息） | 周期内完全连续 | 100%保留 | 100%可复现 | 结构量化分析（唯一选择） |
| 全局前/后复权 | 完全违背（未来信息泄露） | 全局连续但虚假 | 彻底破坏 | 不可复现 | 传统价格比例分析 |
| 原始不复权价格 | 完全符合 | 有跳空不连续 | 被割裂 | 100%可复现 | 无结构分析需求场景 |

#### 4. 除权日指标同步映射（实操关键）
- 映射规则：所有价格类指标（吸引子、P0/P1、贝叶斯分布、止损止盈位）必须与价格路径使用完全相同的除权因子同步缩放
- 实操流程：
  1. 除权日前夜：用交易所官方除权因子预处理所有已输出指标
  2. 除权日开盘：直接使用映射后指标做决策
  3. 双校验：相对比例校验（映射前后涨跌幅一致）、拓扑不变性校验（结构特征无变化）

### （二）tick数据vs低频数据（非硬约束）
#### 1. 核心结论
- 理论层面：结构在不同时间尺度具有严格拓扑同构性，日线/分钟线完全可支撑体系运行
- 实操层面：tick数据是最优选择（提升信噪比与识别精度），但非必须硬约束

#### 2. 不同频率数据适配方案
| 数据频率 | 理论适配性 | 核心优势 | 核心劣势 | 优化方案 |
|----------|------------|----------|----------|----------|
| 毫秒级tick | 完全适配 | 信噪比最高，相变识别精度极高 | 门槛高、存储计算成本大 | 直接使用，无需额外处理 |
| 1/5/15分钟级 | 完全适配 | 门槛低、存储成本小 | 路径细节丢失，抗噪性下降 | 分形插值还原路径，延长结构周期过滤噪声 |
| 小时/日线级 | 完全适配 | 全免费、计算量极小 | 信噪比最低，相变识别延迟 | 结合多维度数据（成交量/资金流），严格过滤短生命周期特征 |

## 三、数据库架构选择与设计
### （一）数据库架构决策
#### 1. 核心结论
- MVP/小实盘阶段：**升级后的DuckDB完全支撑**（单文件数据库，零部署成本）
- 生产级阶段：TimescaleDB（时序）+ Neo4j（图）+ ClickHouse（宽表OLAP）三库组合

#### 2. DuckDB适配性验证（MVP阶段）
| 系统需求 | DuckDB支持情况 | 备注 |
|----------|----------------|------|
| 存储原始K线+除权因子+局部映射价 | 完全支持 | 支持多版本价格列共存 |
| 存储高维结构特征（分形/熵/拓扑指标） | 完全支持 | 列式存储天生适配宽表 |
| 存储吸引子、P0/P1、贝叶斯分布 | 完全支持 | 支持浮点高精度计算 |
| 时序切片查询（回测核心） | 完全支持 | 复合索引优化查询速度 |
| 除权日指标同步映射计算 | 完全支持 | 支持SQL+Python混合计算 |
| 可复现审计日志 | 完全支持 | 支持APPEND ONLY模式 |
| 复杂拓扑关系查询 | 不支持 | 生产级需新增Neo4j |

### （二）DuckDB核心表结构设计（MVP阶段）
#### 1. 局部除权映射价格表（bar_mapped）
```sql
CREATE TABLE IF NOT EXISTS bar_mapped (
    code            VARCHAR,
    interval        VARCHAR,  -- '1d','1m','5m','1h'
    trade_date      DATE,
    bar_time        TIME,

    close_raw       DOUBLE,   -- 原始不复权价格
    rights_factor   DOUBLE,   -- 交易所官方除权因子
    is_ex_rights    BOOLEAN,  -- 是否为除权日

    close_mapped    DOUBLE,   -- 局部除权映射价（结构分析唯一使用价格）

    PRIMARY KEY (code, interval, trade_date, bar_time)
);
```

#### 2. 结构分析结果表（structure_analyze）
```sql
CREATE TABLE IF NOT EXISTS structure_analyze (
    code            VARCHAR,
    interval        VARCHAR,
    trade_date      DATE,
    bar_time        TIME,

    struct_id       VARCHAR,             -- 结构唯一标识
    struct_start_at TIMESTAMP,           -- 结构起始时间
    struct_end_at   TIMESTAMP,           -- 结构结束时间
    struct_type     VARCHAR,             -- 结构类型（趋势/震荡/N字/反转）
    is_closed       BOOLEAN,             -- 是否闭合
    close_type      VARCHAR,             -- 闭合类型（natural/forced）

    -- 吸引子相关
    attractor       DOUBLE,              -- 结构吸引子价格
    attractor_std   DOUBLE,              -- 吸引子标准差

    -- 结构边界相关
    p0              DOUBLE,              -- 反转临界点P0
    p1              DOUBLE,              -- 延续临界点P1

    -- 贝叶斯概率区间
    bayes_mean      DOUBLE,              -- 贝叶斯后验均值
    bayes_lower     DOUBLE,              -- 95%置信区间下限
    bayes_upper     DOUBLE,              -- 95%置信区间上限

    -- 除权映射标记
    rights_factor_used DOUBLE,           -- 使用的除权因子

    PRIMARY KEY (code, interval, trade_date, bar_time)
);
```

#### 3. 结构计算审计日志表（structure_audit）
```sql
CREATE TABLE IF NOT EXISTS structure_audit (
    id              UUID PRIMARY KEY,
    code            VARCHAR,
    interval        VARCHAR,
    bar_time        TIMESTAMP,
    operate         VARCHAR,    -- 操作类型（COMPUTE/MAP_EX_RIGHTS/UPDATE_STRUCT）
    params          JSON,       -- 计算参数、映射记录等
    model_ver       VARCHAR     -- 模型版本
);
```

#### 4. 结构驱动交易信号表（signal_structured）
```sql
CREATE TABLE IF NOT EXISTS signal_structured (
    code            VARCHAR,
    interval        VARCHAR,
    trade_date      DATE,
    bar_time        TIME,

    signal          VARCHAR,    -- 交易信号（LONG/SHORT/EXIT/HOLD）
    trigger         VARCHAR,    -- 触发条件（BREAK_P1/DROP_P0/CLOSED/ATTRACTOR）

    attractor_ref   DOUBLE,     -- 参考吸引子价格
    p_ref           DOUBLE,     -- 参考临界点价格
    prob            DOUBLE,     -- 信号置信度

    PRIMARY KEY (code, interval, trade_date, bar_time)
);
```

## 四、MVP最小可行方案（单机版）
### （一）MVP架构组成
| 组件 | 选型 | 核心作用 | 部署要求 |
|------|------|----------|----------|
| 数据来源 | QMT xtquant API | 获取原始K线、除权因子、逐笔数据 | Python环境+xtquant库 |
| 数据库 | 升级后DuckDB | 存储所有数据、特征、信号 | 单机8G内存即可 |
| 计算引擎 | Python | 结构识别、除权映射、吸引子计算、贝叶斯推断 | 支持GUDHI/PyMC3/GPyTorch库 |
| 执行模块 | 复用现有平台交易接口 | 信号执行、风控、下单 | 对接现有量化平台 |
| 验证模块 | 序贯贝叶斯因子检验 | 实时验证结构有效性 | 嵌入计算引擎 |

### （二）MVP核心工作流程
1. **数据接入阶段**
   - 从QMT拉取目标品种的原始K线（1min/日线）、除权因子
   - 写入bar_mapped表的close_raw和rights_factor字段
   - 标记除权日（is_ex_rights=True）

2. **局部除权映射计算**
   ```python
   def calculate_local_mapped_price(code, interval, start_date, end_date):
       # 1. 读取原始数据
       raw_data = duckdb.execute("""
           SELECT trade_date, bar_time, close_raw, rights_factor, is_ex_rights
           FROM bar_mapped
           WHERE code = ? AND interval = ? AND trade_date BETWEEN ? AND ?
           ORDER BY trade_date, bar_time
       """, [code, interval, start_date, end_date]).fetchdf()
       
       # 2. 局部映射逻辑（仅处理当前周期内除权）
       mapped_prices = []
       current_factor = 1.0
       for idx, row in raw_data.iterrows():
           if row['is_ex_rights']:
               # 除权因子生效（仅影响后续价格）
               current_factor *= row['rights_factor']
           # 计算映射价
           mapped = row['close_raw'] * current_factor
           mapped_prices.append(mapped)
       
       # 3. 更新表中映射价字段
       raw_data['close_mapped'] = mapped_prices
       duckdb.execute("""
           UPDATE bar_mapped
           SET close_mapped = ?
           WHERE code = ? AND interval = ? AND trade_date = ? AND bar_time = ?
       """, [raw_data['close_mapped'], code, interval, raw_data['trade_date'], raw_data['bar_time']])
       
       return raw_data
   ```

3. **结构识别与特征计算**
   - 输入：bar_mapped表的close_mapped字段
   - 计算：分形维数、熵、拓扑贝蒂数（Betti-0/Betti-1）
   - 输出：写入structure_analyze表对应字段

4. **吸引子与边界计算**
   - 用DPMM自动识别吸引子价格及置信区间
   - 用贝叶斯变点检测确定P0/P1临界点
   - 除权日同步映射：吸引子_new = 吸引子_old × 除权因子
   - 结果写入structure_analyze表

5. **信号生成与执行**
   - 触发条件判断：价格突破P1（多）、跌破P0（空）、结构闭合（平仓）
   - 生成信号写入signal_structured表
   - 现有交易平台读取信号表，执行下单逻辑

6. **验证与审计**
   - 每根K线计算贝叶斯因子，验证结构有效性
   - 所有操作记录写入structure_audit表，保证可复现

### （三）MVP验证范围
| 验证内容 | 状态 | 验证方式 |
|----------|------|----------|
| 局部除权映射正确性 | ✅ 可验证 | 对比映射前后相对涨跌幅一致性 |
| 结构拓扑特征稳定性 | ✅ 可验证 | 蒙特卡洛拓扑不变性检验 |
| 除权日指标同步映射 | ✅ 可验证 | 校验映射前后相对位置一致性 |
| 信号生成逻辑完整性 | ✅ 可验证 | 回测中检查信号触发合理性 |
| 全链路可复现性 | ✅ 可验证 | 基于审计日志重建计算过程 |

## 五、现有量化平台嵌入方案（无侵入式升级）
### （一）嵌入原则
1. 不动现有任何架构，仅做底层扩展与字段注入
2. 结构体系作为独立"高级因子模块"接入现有平台
3. 复用现有交易、风控、回测、前端展示能力

### （二）嵌入层级与方式
| 现有平台层级 | 嵌入内容 | 接入方式 |
|--------------|----------|----------|
| 数据层 | 新增4张DuckDB表 | 独立建表，不修改原有表结构 |
| 计算层 | 结构分析模块（除权映射、结构识别、吸引子计算） | 封装为Python函数库，接入现有策略框架 |
| 策略层 | 结构驱动策略模板 | 作为新策略类型，调用结构分析模块输出 |
| 信号层 | 结构信号表（signal_structured） | 现有信号执行引擎读取该表 |
| 回测层 | 结构回测适配器 | 适配现有回测框架，加载结构特征数据 |
| 审计层 | 结构计算审计日志 | 接入现有审计系统，支持回溯 |

### （三）嵌入后平台能力升级
| 原有能力 | 升级后能力 | 核心差异 |
|----------|------------|----------|
| 基于指标/均线的策略 | 基于拓扑结构的策略 | 从"表象映射"到"本质识别" |
| 固定参数模型 | 动态开放模型 | 从"封闭拟合"到"开放演化" |
| 全局复权数据处理 | 局部除权映射 | 从"破坏因果"到"保持拓扑" |
| 固定样本回测 | 序贯动态验证 | 从"历史拟合"到"科学验证" |
| 确定性信号 | 概率化信号 | 从"非黑即白"到"风险量化" |

## 六、关键约束与风险防控
### （一）必须遵守的硬约束
1. 数据层：原始数据只读，所有映射操作仅在内存或新增字段中执行
2. 计算层：禁止使用未来信息，除权因子仅在除权日之后生效
3. 策略层：结构分析必须基于close_mapped字段，禁止直接使用复权价或原始价
4. 验证层：必须分层证伪，先校验辅助假设，再校验核心理论
5. 审计层：所有操作必须记录日志，保证100%可复现

### （二）常见风险与防控
| 风险类型 | 风险描述 | 防控措施 |
|----------|----------|----------|
| 拓扑结构失真 | 复权或数据处理破坏路径连续性 | 强制使用局部除权映射，双校验拓扑不变性 |
| 未来信息泄露 | 除权因子提前生效 | 严格按时间顺序处理，除权因子仅影响后续价格 |
| 模型封闭固化 | 固定结构参数导致失效 | 使用贝叶斯非参数框架，动态扩展参数空间 |
| 信号误触发 | 噪声导致虚假突破/跌破 | 结合多重分形过滤，延长特征生命周期阈值 |
| 回测过拟合 | 固定样本优化参数 | 采用序贯回测，禁止样本内优化 |

## 八、核心结论
